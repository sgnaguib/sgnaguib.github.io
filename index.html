Kinector Part I

The most recent major web development project I completed was Kinector, a web app commissioned by UBC Neurology professor, Dr. Steven Pelech. I worked in a group of four to develop this website from the ground up and now it’s used by researchers to world over to help them visualize the interactions between the thousands of proteins found in human cells. Graphs from the web app will even be used in a soon-to-be-published research paper by Dr. Pelech. The first part of this project involved generating graphs for the direct connections between proteins.

Kinector Part II

As a follow-up project for Dr. Pelech, I helped develop a generator for graphs of the indirect connections between proteins. For example, one graph may show that protein A activates protein B, which in turn activates protein C. This is a one-of-a-kind tool for helping molecular biologist study the indirect relationships between proteins.

HaikuArt

Aside from web development, I’ve also done mobile development. I’ve just finished developing HaikuArt, an iOS app that maps the public artwork in the city of Vancouver and allows users to read and write haikus about them. I’m planning on submitting the app to the App Store by the end of October.

KnowledgeBase

Regarding backend web development, this past spring I was part of a team that developed a Stack Overflow lookalike website. We made a fully functioning platform where users could create profiles, post and reply to questions and view the profiles of other users. Through that project, I learnt to implement a MVC architecture and to do backend development with Node.js and a PostgreSQL database.

Allegro

Another website I’ve developed, a bit more original, is Allegro, a web app game for learning to read music notes. For the website, I used vexflow.js to render the music notes and Google’s Firebase to store all user high scores. The website proved useful as music teachers now assign the game to their beginner students.

Todos

Most recently, I built a simple todo web app as an exercise in learning the fundamentals of Angular. A user can add tasks to the list, delete them, or mark them as complete. The list's data is stored in a mongoDB database, which is accessed and updated using Node.js.
